# 钉钉企业内部机器人  接入大模型

基于LangChain的钉钉企业内部机器人，接入大模型，提供智能问答、自动化任务等功能。


## 开发流程

### Webhook 推送机器人

起初仅想实现一个简单的Webhook推送机器人，通过爬取新闻，经过大模型分析总结，推送到群里

实现后，感觉没有灵魂，仅有总结部分使用到了大模型，不够智能

#### 实现步骤

首先自行搭建LangChain的API调用框架

通过Claude Code（minimax m-2.5）实现定时任务模块，新闻爬取出现问题，无法正确爬取内容

去minimax官网，使用minimax m-2.7成功生成了爬取新闻的代码，成功爬取到内容

手动整合爬取新闻和总结的代码，成功实现了定时爬取新闻并总结的功能

### 对话机器人

在Webhook推送机器人的基础上，想要实现一个对话机器人，用户可以在群里直接和机器人对话，下达指令，机器人可以调用工具完成任务

#### 实现步骤

首先去钉钉开发者平台，创建一个企业内部机器人，获取机器人id与秘钥，以及开通机器人权限

尝试使用Claude Code（minimax m-2.5）在现有代码基础上实现对话功能，效果不佳

去minimax官网，使用minimax m-2.7 让其去钉钉开发者平台查看文档，再进行代码生成，依旧失败

自行去钉钉开发者平台查看文档，了解了钉钉机器人相关的接口  发现钉钉提供的SDK接口十分复杂

消息接口 包括新版SDK 旧版SDK 以及使用cool应用 不使用cool应用 等等，交给AI阅读十分困难

最终自行找到了新版不使用cool应用的消息接口，成功实现了对话功能

在此基础上，给AI提供了定时任务管理的工具，成功实现了用户可以通过对话来管理定时任务的功能

由此 可以实现如“30分钟后提醒xxx开会”的功能，AI会调用工具来创建一个定时任务，30分钟后推送消息提醒开会

AI管理定时任务实现：
- （定时任务需要一个具体任务函数）添加一个通用job任务函数，函数功能为向AI发送一条指令，指令内容为参数
    - 通用的job任务函数可以满足不同指令的定时执行需求
    - 这一种函数均为给AI发送指令  还可以继续添加其他类型的通用任务函数
- 给AI增加添加任务工具，添加任务（实际上是调用通用任务函数，以不同的输入指令为参数）
- 删除任务工具，调用调度器的删除任务接口
